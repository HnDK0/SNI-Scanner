# SNI VLESS Scanner

Инструмент для поиска сайтов в одной подсети или ASN с целевым IP-адресом,
подходящих для использования в качестве SNI-домена в конфигурации VLESS.

---

## Что такое SNI в контексте VLESS

**VLESS + TLS + SNI** — схема, при которой клиент устанавливает TLS-соединение
с сервером, указывая в поле `serverName` (SNI) домен легитимного сайта.
Это позволяет трафику выглядеть как обычный HTTPS к известному сервису.

Чтобы SNI работал корректно, нужен домен, который:
- находится на том же IP или в той же сети, что и ваш сервер (или CDN-узел)
- поддерживает **TLS 1.2 / 1.3**
- желательно отдаёт **HTTP/2** (ALPN `h2`) — так трафик лучше маскируется

Именно такие домены и ищет этот скрипт.

## Установка

### Требования

- Python 3.9+
- Библиотека `requests` (опционально, но ускоряет работу)

```bash
pip install requests
```

### Загрузка

```bash
# Скачать скрипт
curl -O https://your-host/sni_scanner.py
# или просто скопируйте файл sni_scanner.py в рабочую папку
```

---

## Быстрый старт

```bash
python3 sni_scanner.py 1.2.3.4
```

Скрипт автоматически:
1. Определит подсеть `/24` и ASN по IP
2. Соберёт все IP из подсети и префиксов ASN
3. Просканирует их на наличие TLS/SNI
4. Выведет список кандидатов и сохранит результат в `.txt`

---

## Параметры запуска

| Параметр | По умолчанию | Описание |
|---|---|---|
| `ip` | — | Целевой IP-адрес (обязательный) |
| `--mode` | `both` | Режим: `subnet`, `asn`, или `both` |
| `--subnet-prefix` | `24` | Длина маски подсети (`/24`, `/22`, и т.д.) |
| `--threads` | `50` | Число параллельных потоков |
| `--timeout` | `3.0` | Таймаут соединения в секундах |
| `--port` | `443` | TLS-порт |
| `--max-hosts` | `1000` | Максимум IP из ASN для сканирования |

### Примеры

```bash
# Только подсеть /24
python3 sni_scanner.py 104.21.5.100 --mode subnet

# Только ASN, до 2000 хостов, 100 потоков
python3 sni_scanner.py 104.21.5.100 --mode asn --max-hosts 2000 --threads 100

# Расширенная подсеть /22 с быстрым таймаутом
python3 sni_scanner.py 104.21.5.100 --mode subnet --subnet-prefix 22 --timeout 2

# Нестандартный порт TLS
python3 sni_scanner.py 104.21.5.100 --port 8443
```

## Как работает сканирование

```
Целевой IP
    │
    ├─► ipinfo.io → ASN, страна, организация
    │
    ├─► Подсеть /24 → список IP
    │
    └─► bgpview.io → префиксы ASN → список IP
             │
             ▼
       Для каждого IP:
         1. Обратный DNS (rDNS) → hostname
         2. TLS без SNI → CN из сертификата
         3. TLS с SNI (hostname) → версия TLS, ALPN
             │
             ▼
       Фильтр: TLS 1.2/1.3 ✓  →  результат
               HTTP/2 (h2)  ✓  →  приоритет
```

## Применение результатов в VLESS

После получения списка кандидатов возьмите любой домен с пометкой `[H2]`
и укажите его в конфиге клиента (Xray / sing-box / v2rayN и т.д.).

### Xray (config.json)

```json
{
  "outbounds": [{
    "protocol": "vless",
    "settings": {
      "vnext": [{
        "address": "ВАШ_СЕРВЕР_IP",
        "port": 443,
        "users": [{ "id": "UUID", "encryption": "none" }]
      }]
    },
    "streamSettings": {
      "network": "tcp",
      "security": "tls",
      "tlsSettings": {
        "serverName": "example.com",
        "allowInsecure": false
      }
    }
  }]
}
```

### sing-box (config.json)

```json
{
  "type": "vless",
  "server": "ВАШ_СЕРВЕР_IP",
  "server_port": 443,
  "uuid": "UUID",
  "tls": {
    "enabled": true,
    "server_name": "example.com",
    "insecure": false
  }
}
```

> **Важно:** `serverName` / `server_name` — это домен из результатов скрипта.
> `address` / `server` — это IP вашего VPS/сервера, **не** найденный IP.

## Советы и рекомендации

**Выбирайте домены с HTTP/2.** Трафик h2 значительно лучше имитирует
обычный браузерный HTTPS и меньше привлекает внимание DPI-систем.

**Предпочитайте крупные CDN.** Домены на Cloudflare, Fastly, Akamai
находятся на общих IP с тысячами других сайтов — это дополнительная
маскировка («IP в диапазоне Cloudflare» не вызывает подозрений).

**Проверяйте стабильность.** Некоторые сайты меняют IP или переезжают.
Запускайте скрипт повторно раз в несколько недель.

**Увеличьте --max-hosts для ASN.** Если кандидатов мало, попробуйте
`--max-hosts 3000` — у крупных провайдеров сотни тысяч адресов.

**Снизьте таймаут для скорости.** `--timeout 1.5 --threads 150` ускорит
сканирование примерно втрое, но может дать больше ложных промахов.

---

## Устранение проблем

**Нет результатов из ASN**
Скрипт обращается к `bgpview.io`. Если API недоступен, используйте
`--mode subnet` или проверьте подключение.

**Мало кандидатов в подсети /24**
Попробуйте расширить подсеть: `--subnet-prefix 22` охватит 1022 IP.

**Зависание на больших ASN**
Ограничьте выборку: `--max-hosts 500 --timeout 2`.

**ModuleNotFoundError: requests**
Скрипт работает и без `requests` (через `urllib`), но для установки:
```bash
pip install requests --break-system-packages
```

## Структура файла результатов

Файл `sni_results_*.txt` содержит строки формата:

```
IP <TAB> hostname <TAB> TLS-версия <TAB> h2|h1 <TAB> cert-CN
```

Пример:
```
104.21.5.12    example.com    TLSv1.3    h2    example.com
104.21.5.88    oldsite.org    TLSv1.2    h1    oldsite.org
```

## Зависимости

| Библиотека | Тип | Назначение |
|---|---|---|
| `socket` | встроенная | TCP-соединения |
| `ssl` | встроенная | TLS-рукопожатие, чтение сертификатов |
| `ipaddress` | встроенная | Работа с подсетями и диапазонами IP |
| `concurrent.futures` | встроенная | Параллельное сканирование |
| `requests` | опциональная | HTTP-запросы к ipinfo.io / bgpview.io |
| `urllib` | встроенная | Резервный HTTP-клиент (если нет requests) |

## Дисклеймер

Скрипт выполняет только пассивное зондирование: устанавливает TCP+TLS
соединение и читает публично доступный сертификат. Тем не менее убедитесь,
что его использование соответствует законодательству вашей страны и
политике провайдера.
